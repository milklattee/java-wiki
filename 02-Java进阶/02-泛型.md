# 泛型 (Generics)

## 为什么需要泛型

- **类型安全** — 编译期检查，避免 `ClassCastException`
- **代码复用** — 一套代码支持多种类型
- **消除强制转换** — 无需手动 cast

## 泛型类

```java
public class Box<T> {
     private T value;
     public void set(T value) { this.value = value; }
     public T get() { return value; }
}

Box<String> box = new Box<>();
box.set("hello");
String s = box.get();   // 无需转型
```

## 泛型方法

```java
public static <T> T identity(T value) {
     return value;
}

String result = Util.<String>identity("hello");
// 类型推断可省略:
String result = Util.identity("hello");
```

## 通配符

| 通配符 | 含义 | 用途 |
|--------|------|------|
| `?` | 未知类型 | `List<?>` 只读 |
| `? extends T` | T 或 T 的子类 | **上界** — 生产者 (读取) |
| `? super T` | T 或 T 的父类 | **下界** — 消费者 (写入) |

记忆口诀：**PECS** — Producer Extends, Consumer Super

```java
// 可读不可写
void printList(List<? extends Number> list) {
     for (Number n : list) System.out.println(n);
}

// 可写不可读
void addNumbers(List<? super Integer> list) {
     list.add(42);
}
```

## 类型擦除

泛型信息在编译后擦除，运行时不存在。因此：
- 不能用 `instanceof` 检验泛型类型
- 不能创建泛型数组 `new T[10]`
- 静态上下文中不能使用类型参数
